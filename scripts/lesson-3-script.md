La siguiente sección sería la sección de párrafos. En este caso, podemos ver cómo se realizan los párrafos solamente con la etiqueta <p>. Dentro de estos, podemos utilizar la etiqueta <strong> para hacer negrita, o la etiqueta <em> para hacer cursiva dentro de este mismo párrafo. Cada párrafo tiene una separación del anterior, por defecto, en HTML, tanto antes como después. Esto no influye a la hora de escribirlo en el código, sino a la hora de presentarse. Finalmente, tenemos lo que serían las citas, que es parecido pero con unas llaves al inicio. Fácilmente podría escribirse aquí todo un párrafo, no afecta el límite en nada. Y ahora que estamos viendo el código, cabe mencionar que <hr> permite generar una línea de separación.

Ahora iríamos con lo que son listas. Esto se utiliza, sobre todo, para generar lo que son top headers, como este de acá, a la hora de querer mostrar distintas opciones. Es bastante intuitivo. En primer lugar, tendríamos listas desordenadas, que enlistan los elementos a través de puntitos y no tienen un orden específico. Se realiza con la etiqueta <ul> y, dentro de ella, cada elemento listado debe estar con <li>. Si se realiza, por ejemplo, con <p>, no se enlista correctamente. Lo mismo pasa con las listas ordenadas, que son exactamente lo mismo, solo que con numeración numérica, valga la redundancia.

Ahora, en lo que sería la sección de imágenes, tenemos una imagen de ejemplo. Esta es una imagen al azar que elegí de internet, pero se puede seleccionar cualquier imagen a través de su link de código fuente, usando la etiqueta <img> con el atributo src (source) igual al link de lo que queremos añadir. En este editor de código no me permite subir imágenes sin pagar, así que por temas de ejemplo he utilizado una imagen de internet. Sin embargo, en la mayoría de proyectos uno puede trabajar subiendo sus propios archivos. Algo así es como debería verse en proyectos regulares que no utilicen recursos de internet.

El atributo alt sirve para mostrar un texto por si no llega a cargar la imagen, por ejemplo, "Texto alternativo a la imagen". Esto se suele utilizar por errores de conexión, más que nada. También, desde la propia etiqueta se puede controlar el ancho y el alto. Si modificamos el alto a 30, controlamos el límite de altura. Como no estaba especificado antes, solo le decíamos el límite de ancho y la altura se ajustaba de forma automática. Ahora, como ya tiene los dos valores fijos, se fuerza a la imagen a seguir ese patrón. Como se han dado cuenta, la etiqueta <img> es autocontenida, no necesita un cierre porque no requiere ningún texto dentro de ella.
